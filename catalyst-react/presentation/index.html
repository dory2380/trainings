Why React?

  • It's incredibly popular
  • There is a large developer community
  • You can transition to native mobile apps with only a small learning curve
  • The performance is pretty good
  • Development is backed by a company with deep pockets

What is React?

  • It is a JavaScript views library - not a framework
  • Allows us to create reuseable user interface components
  • We bolt other libraries onto React to provide more functionality

Common stack

  • ES6 (JavaScript)
  • React
  • React Router
  • Redux
  • Thunk
  • Jest

Components

Components let you split the UI into independent, reusable pieces, and think about each piece in isolation.

Conceptually, components are like JavaScript functions. They accept arbitrary inputs (called “props”) and return React elements describing what should appear on the screen.

reactjs.org

Our app - Component Structure

Props

Props are how components talk to each other

Props are passed from a parent component to a child component and are immutable

State

State is managed within the component and unlike props, is mutable

We will go into more detail about both props and state as we build our application

Other things of note...

ES6

  • Instead of var we use let and const
  • Functions written as arrow functions 
    
                    (param1, param2) => {
                      // body of function goes here
                    }
  • We can import from other files using modules
  • Classes are easy to create

JSX

JSX is a syntax extendion to JavaScript and is the recommended way to describe what the UI should look like in React

  • Allows us to combine our logic and markup in one file
  • Embed JavaScript expressions with curly braces ({})
  • Since JSX is closer to JavaScript then html we use the camelCase naming convention, for example className instead of class
